Cloud Sync & Backup Offline only

Connect your own free Supabase database to store your data in the cloud — so it's backed up, safe if your browser is cleared, and available when you log in from another computer. This is optional: until you connect and log in, the app keeps working exactly as now, saving to this browser. Show setup steps
One-time setup (about 5 minutes):
1. Go to supabase.com → sign up (free) → New project. Give it a name and a database password, pick a region near you, and wait ~2 min for it to finish.
2. In your project, open Project Settings → API. Copy the Project URL and the anon public key. Paste both into the boxes below.
3. Open the SQL Editor (left menu) → New query, paste the snippet below, and click Run. This creates the tables that hold your data (clients, invoices, audit log, settings, users) — one row per record, so different devices editing different records never conflict — and locks each table down so each account only sees its own data.
4. (Recommended) In Authentication → Providers → Email, turn off "Confirm email" while testing, so you can log in immediately without clicking a confirmation link.
5. (Required for instant sync) Go to Database → Replication, and toggle on the clients, invoices, and audit_log tables. This makes changes appear on other devices within a second or two. If you skip this, syncing still works — it just checks for updates automatically every ~8 seconds instead of instantly.
6. Come back here, paste your URL + key, click Save & Connect, then create your account below.

Security & Access Demo OTP

Manage login security for this app. Add teammates and set exactly what each one can see and do from the Users tab. Configure how one-time codes are delivered below, and manage your own password/2FA from your profile menu (top-right).
Login data (password hashes, roles) is stored in this browser's local storage, same as the rest of your app data — it is not sent anywhere unless you connect Cloud Sync above or enable live OTP delivery. Passwords are hashed (SHA-256 + per-user salt), never stored in plain text, but this is a client-side app and not a substitute for a real backend when protecting highly sensitive data.
